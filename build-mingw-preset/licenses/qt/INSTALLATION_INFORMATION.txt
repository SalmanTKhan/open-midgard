OpenMidgard's Qt integration is deployed using shared Qt DLLs and plugins.

For Qt-enabled builds, the executable is expected to load Qt from the copied runtime files placed next to the application by `windeployqt`.

To test a modified Qt build against this application:

1. Keep the OpenMidgard executable unchanged.
2. Replace the deployed Qt DLLs, plugins, and QML runtime files with a modified, interface-compatible Qt build of the same major/minor line.
3. Launch the application normally from the deployed folder.

No device lock, signature gate, or encrypted packaging step is added by this project to prevent replacing the Qt runtime files in the deployed folder.

If a future release adds installer logic, code signing, or packaging changes that affect runtime replacement, this file should be reviewed so the installation information remains accurate.
